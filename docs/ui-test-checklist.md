# anoon web — Чеклист ручного UI-теста

> Прогонять на `localhost:3000` (desktop) и на реальном устройстве. Легенда:
> ✅ должно работать · ⚠️ ограничено окружением · 🔵 требует бэкенда/тумблера.
> Запуск: `pnpm --filter webapp dev`.

## Окружение
- [ ] Desktop Chrome/Edge (`http://localhost:3000`)
- [ ] Телефон в той же сети (`http://<LAN-IP>:3000`) — ⚠️ голос/push не работают без HTTPS
- [ ] Установка PWA (Add to Home Screen) — standalone, без адресной строки

## 1. Онбординг
- [ ] Первый заход без профиля → экран онбординга (лого «a», поле ник)
- [ ] Кнопка «Начать общение» неактивна при нике <2 символов
- [ ] Ввод ника ≥2 → кнопка активна
- [ ] Сабмит → 🔵 при вкл. anonymous auth «Входим…» и вход; сейчас фолбэк → сразу на поиск
- [ ] После создания профиля reload → сразу экран поиска (persist сессии)

## 2. Экран поиска
- [ ] Отображается ник + #ID (glass-карточка), бренд-свечение фон
- [ ] Иконка настроек (шестерёнка) → переход на /settings
- [ ] Push-тумблер (колокол) виден
- [ ] Кнопка «Найти» → спиннер «Ищем…» → 🔵 переход в чат (сейчас мок: случайный peer)
- [ ] Тач-цели ≥44px (шестерёнка, push, «Найти»)

## 3. Чат — текст
- [ ] Хедер: «Собеседник» + #peer, зелёная точка онлайн, меню «⋮»
- [ ] Кнопка «Назад» → на поиск
- [ ] Ввод текста + Enter → сообщение справа (своё), автоскролл вниз
- [ ] 🔵 автоответ «Интересно 🙂» (мок; уйдёт с Realtime)
- [ ] Reload → история чата сохранилась (persist)

## 4. Чат — медиа
- [ ] Кнопка «прикрепить» (картинка) → выбор файла фото/видео
- [ ] После выбора → превью-бар (миниатюра + имя + «отправить»/«убрать»)
- [ ] Отправка фото → пузырь-картинка; тап → полноэкранный лайтбокс
- [ ] Лайтбокс: закрытие по X / Esc / тап по фону / свайп вниз
- [ ] Отправка видео → пузырь с кнопкой play + длительность; тап → лайтбокс (видео, controls)
- [ ] ⚠️ Запись голоса (микрофон): на localhost работает; по LAN-IP → баннер «Нет доступа к микрофону»
- [ ] Голос: запись → таймер → отправить (✓) / отменить (✗); воспроизведение с прогрессом
- [ ] Reload → медиа-сообщения показывают плашку «Медиа недоступно» (blob-URL мёртв)

## 5. Меню чата (⋮) — модерация
- [ ] «⋮» → меню «Пожаловаться» / «Заблокировать»
- [ ] «Пожаловаться» → диалог: выбор причины (радио) + коммент → «Отправить» → «Спасибо»
- [ ] «Заблокировать» → confirm → выход на / ; 🔵 (сейчас локальный стор)
- [ ] Заблокированного нельзя открыть повторно (`/chat/<id>` → редирект на /)

## 6. Настройки (/settings)
- [ ] Смена ника → «Сохранить» → «Готово»; reload → ник сохранён; #ID не меняется
- [ ] Push-секция: тумблер + статус; при недоступности — видимая ошибка
- [ ] Список заблокированных: показ #ID + «Разблокировать» → исчезает из списка
- [ ] «Выйти» → сброс профиля → онбординг

## 7. PWA / офлайн
- [ ] manifest подхватывается (иконки 192/512, standalone, #000)
- [ ] Иконки на домашнем экране корректные (не битые)
- [ ] 🔵 Офлайн-режим (после Serwist precache) — оболочка грузится без сети

## 8. Push (🔵 требует HTTPS + тумблер)
- [ ] Тумблер → запрос разрешения браузера
- [ ] Разрешено → подписка создаётся (VAPID public задан)
- [ ] ⚠️ по LAN-IP без HTTPS → ошибка/недоступно (ожидаемо)
- [ ] 🔵 реальная доставка push (нужен backend-рассыл)

## 9. Тема / A11y / отзывчивость
- [ ] Фон #000, акцент #FDBF2D везде
- [ ] Контраст текста читаем (AA)
- [ ] Фокус-обводка на интерактиве (Tab-навигация)
- [ ] `prefers-reduced-motion` → анимации отключаются/укорачиваются
- [ ] Верстка 375px (iPhone SE) … 430px (макс телефон) без горизонт. скролла
- [ ] Safe-area (вырез/индикатор) не перекрывает контент в standalone

## Известные ограничения (не баги)
- Чат/матчинг/медиа-отправка — моки до Realtime/R2 (Фазы D/E).
- Голос/Push требуют secure-context (localhost или HTTPS).
- Anonymous auth выключен → сессия в фолбэк-моке (Фаза C).
