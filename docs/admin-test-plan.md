# anoon admin — Тест-план (безопасность + модерация)

> Из ревью `admin/PLAN.md`. Прогонять когда admin переключён на реальный API + admin-auth
> (Фаза F). Легенда: 🔴 критично (безопасность) · 🟠 важно · ⚪ качество.

## 1. Аутентификация admin 🔴
- [ ] Нет токена → любой admin-эндпоинт = 401 (default-deny middleware).
- [ ] Юзерский JWT (anon) НЕ принимается для admin (разные audience/issuer/ключ).
- [ ] Чужой/просроченный/подделанный токен → 401.
- [ ] Логин: неверный пароль → 401, без утечки «есть ли такой email».
- [ ] Пароль хранится argon2id (не bcrypt/plain); проверить формат хеша в БД.
- [ ] 2FA (TOTP): без кода → отказ; неверный код → отказ; окно времени соблюдается.
- [ ] Cookie сессии httpOnly + Secure + SameSite; недоступна из JS.
- [ ] Rate-limit на логин (N попыток → throttle/лок).

## 2. Авторизация / RBAC 🔴
- [ ] `moderator` не может: создавать AdminUser, менять роли, видеть настройки super-admin.
- [ ] `super_admin` имеет полный доступ.
- [ ] RBAC проверяется на уровне API (не только скрытием UI-кнопок).

## 3. Приватность / PII 🔴
- [ ] Списки/детали НЕ возвращают emailHash, providerId, IP по умолчанию.
- [ ] Просмотр приватного контента — только по конкретной жалобе, с причиной.
- [ ] Каждый просмотр приватного → запись ModeratorAction(view_private) в аудит.
- [ ] CSAM/illegal: не даёт свободного просмотра, только эскалация (блокировка элемента).

## 4. Бан / разбан 🟠
- [ ] Бан создаёт строку Ban(active) + причина + срок; идемпотентность (повторный бан → 409/no-dup).
- [ ] Разбан → Ban.state=lifted + liftedBy/liftedAt; реверс эффекта на юзера.
- [ ] Истечение срока → state=expired (крон/ленивая проверка).
- [ ] Каскад: бан отражается на профиле (не может слать сообщения).
- [ ] Реверс репутации/reportCount при разбане корректен (без двойного счёта).

## 5. Mute 🟠
- [ ] Mute → mutedUntil; enforcement на WS (сообщения не проходят до срока).
- [ ] Снятие/истечение mute восстанавливает отправку.

## 6. Жалобы (lifecycle) 🟠
- [ ] open → in_review (assignedToId) → resolved_banned/resolved_dismissed.
- [ ] Два модератора не разбирают одну жалобу одновременно (assignedTo блокирует).
- [ ] Разрешение пишет resolvedById/resolvedAt + ModeratorAction.
- [ ] targetMessageId/targetMediaId ведут к scoped-просмотру.

## 7. Аудит-полнота 🔴
- [ ] КАЖДАЯ мутация (ban/unban/mute/dismiss/escalate) → строка ModeratorAction.
- [ ] КАЖДЫЙ просмотр приватного → строка (view_private).
- [ ] Логин админа → строка (login) с ip.
- [ ] Аудит неизменяем из UI (нет delete/update).

## 8. Медиа-модерация 🟠
- [ ] Медиа отдаётся через прокси admin-API (не прямой R2/presigned наружу).
- [ ] blur-by-default; «показать» логируется как view_private.
- [ ] «Медиа удалено/истекло» (deletedAt/expiresAt) — корректная плашка.
- [ ] Escalate(CSAM) блокирует элемент + аудит, без просмотра.

## 9. Данные / целостность ⚪
- [ ] reportCount денормализован, без N+1 в списке юзеров.
- [ ] Cursor-пагинация на больших списках.
- [ ] Каскадные удаления (onDelete) не оставляют сирот.

## 10. E2E-сценарии (после Фазы F) ⚪
- [ ] Жалоба из web → появляется в очереди admin.
- [ ] Бан из admin → юзер в web не может писать / кикнут.
- [ ] Разблокировка/разбан → восстановление.
